## 2024-05-15 - Optimizing Readability Metric Parsing
**Learning:** `analyzeText` in `apps/quill/src/lib/rubric.ts` executes on every keystroke in the playground to display the readability metrics (like FK grade, average sentence length, etc.). The syllable counting loop inside `analyzeText` heavily utilizes regex patterns to count vowels to estimate syllable counts for words, which caused serious main-thread blocking when working with large texts. Since this function is executed continuously and the text sizes can be large, executing regex tests over and over again on words inside a loop caused performance bottlenecks.
**Action:** Adding a simple `syllableCache = new Map<string, number>()` to cache syllable lengths of words reduced execution time by 3-4x. Also combining the word loop inside `analyzeText` reduces mapping time. Always consider caching expensive regex executions inside a hot path (e.g. `analyzeText`), and ensure there is only one map/filter over the same array whenever possible.
